# 테이블 뒤 콘텐츠 병합 버그 테스트

이 문서는 테이블 뒤에 오는 다양한 블록 타입이 테이블 안으로 병합되지 않는지 검증합니다.

## 케이스 1: 테이블 → 인용문

| 이름 | 역할 |
|------|------|
| Alice | 개발자 |
| Bob | 디자이너 |

> 이것은 테이블 뒤의 인용문입니다.
> 테이블 안으로 들어가면 안 됩니다.

## 케이스 2: 테이블 → 코드블록

| 명령어 | 설명 |
|--------|------|
| git add | 스테이징 |
| git commit | 커밋 |

```javascript
// 이 코드블록은 테이블 밖에 있어야 합니다
console.log("테이블 뒤 코드블록");
```

## 케이스 3: 테이블 → 중첩 리스트

| 항목 | 수량 |
|------|------|
| 사과 | 10 |
| 바나나 | 5 |

- 과일 목록
  - 사과: 빨간색
  - 바나나: 노란색
    - 미니 바나나
    - 일반 바나나
- 채소 목록
  - 당근

## 케이스 4: 테이블 → 일반 텍스트 → 테이블

| A | B |
|---|---|
| 1 | 2 |

이 텍스트는 두 테이블 사이에 있습니다.

| C | D |
|---|---|
| 3 | 4 |

## 케이스 5: 테이블 → 구분선 → 인용문

| X | Y |
|---|---|
| a | b |

---

> 구분선 뒤의 인용문

## 검증 완료

모든 케이스에서 테이블 뒤 콘텐츠가 올바르게 분리되어야 합니다.
